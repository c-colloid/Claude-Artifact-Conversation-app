## 🚧 実装中

### 【提案1】既存キャラクター自動セットアップ機能

**概要**: アニメ・漫画・ゲームなどの既存キャラクター名を入力すると、Web検索で情報を収集し、自動的にキャラクター設定をセットアップ

**実装内容**:
- キャラクター名 + 作品名を入力
- WebSearchで以下を収集:
  - 性格、口調、一人称/二人称
  - 背景ストーリー
  - 決め台詞・口癖
  - 人間関係
- Claude APIで情報を整理してキャラクター定義に変換
- プレビュー画面で確認・編集後に保存

**メリット**:
- ロールプレイ好きなユーザーが簡単に始められる
- 好きなキャラクターとすぐに会話できる
- 設定を考える手間が省ける

**実装日**: 2025-11-21（予定）

## 📋 将来実装を検討する機能

### 【提案2】AIキャラクター生成アシスタント

**概要**: 短い説明文から、詳細なキャラクター設定を自動生成

**使用例**:
```
入力: "明るくて元気な女子高生、料理が得意"
↓
自動生成:
- 性格: 社交的で前向き、面倒見が良い
- 口調: 「〜だよ！」「〜だね」といったフレンドリーな口調
- 一人称: 「私」
- 二人称: 「あなた」「〇〇くん/ちゃん」
- 背景: 料理部に所属、将来はパティシエを目指している...
- 決め台詞: 「今日も元気にいこー！」「私の特製料理、食べてみる?」
```

**実装内容**:
- 簡単な説明入力フィールド
- 「詳細設定を生成」ボタン
- 生成された設定をプレビュー・編集可能

**メリット**:
- オリジナルキャラクターを簡単に作成できる
- 詳細な設定を考える手間が省ける
- 初心者でも魅力的なキャラクターが作れる

**想定ユーザー**: オリジナルキャラクターで遊びたいユーザー

### 【提案3】対話式キャラクター作成ウィザード

**概要**: AIと対話しながら、質問に答える形でキャラクターを作成

**フロー例**:
```
AI: 「どんなキャラクターを作りますか？」
ユーザー: 「クールな男性キャラ」

AI: 「年齢や職業はどうしますか？」
ユーザー: 「25歳くらいの刑事」

AI: 「性格についてもう少し詳しく教えてください」
ユーザー: 「冷静だけど実は情に厚い」

AI: 「なるほど！口調はどんな感じがいいですか？」
...
```

**実装内容**:
- ステップバイステップのウィザードUI
- AIとの対話形式のインターフェース
- 各ステップで質問と回答を繰り返す
- 最後に生成された設定をプレビュー

**メリット**:
- 初心者でも迷わずキャラクター作成できる
- 対話形式で楽しくキャラクターを作れる
- どんな項目があるか分からない初心者に優しい

**想定ユーザー**: キャラクター作成が初めてのユーザー、何を設定すればいいか分からないユーザー

---

### 【提案4】キャラクター関係性自動提案

**概要**: 複数のキャラクターを選択すると、それぞれの設定から自然な関係性を自動提案

**実装内容**:
- 2人以上のキャラクターを選択
- 各キャラの性格・背景から相性を分析
- 関係性（友人、ライバル、恋人など）を複数提案
- 関係性の説明文も自動生成

**例**:
```
キャラA: 明るく社交的な女子高生
キャラB: 無口でクールな男子生徒

提案:
1. 「幼馴染」- 対照的な性格だが長年の付き合い
2. 「クラスメイト」- Aが一方的に話しかける関係
3. 「部活仲間」- Aがムードメーカー、Bは実力者
```

**メリット**:
- キャラクター同士の関係性を考える手間が省ける
- マルチキャラクターアプリならではの機能
- より自然な会話が生まれる

**想定ユーザー**: 複数キャラクターで遊ぶユーザー、ストーリー性を重視するユーザー

**既存機能との関連**: 現在の関係性システム（Multi character chat.jsx:3607）を拡張

### 【提案5】会話から学習・成長するキャラクター

**概要**: 会話履歴を分析し、キャラクターの新しい側面を発見・提案

**実装内容**:
- 定期的に会話履歴を分析
- 「このキャラクターはこんな一面もありそうです」と提案
- 新しい決め台詞、性格の追加要素を提案
- ワンクリックで設定に追加可能

**例**:
```
通知: 「太郎は料理の話題でよく盛り上がっていますね。
      背景に『実は料理が趣味』を追加しますか？」
```

**実装内容の詳細**:
- 一定数のメッセージごとに会話を分析
- よく出てくる話題やキャラクターの反応パターンを抽出
- 新しい性格要素、決め台詞、背景情報を提案
- 通知 or モーダルで提案を表示
- 承認するとキャラクター設定に自動追加

**メリット**:
- キャラクターが会話を重ねるごとに深みが出る
- ユーザーとの会話で自然に成長する
- 長く使うほど愛着が湧く

**想定ユーザー**: 長期的にキャラクターと会話を楽しむユーザー

**技術的考慮点**:
- 会話履歴の分析頻度とコスト
- プライバシーへの配慮
- 適切な提案のタイミング

### 【提案6】キャラクターテンプレートライブラリ

**概要**: よくあるキャラクタータイプのテンプレートを用意

**カテゴリ例**:
- **性格系**: ツンデレ、クーデレ、ヤンデレ、天然、元気っ子、大人しい子、etc.
- **職業系**: 教師、医者、騎士、魔法使い、探偵、アイドル、etc.
- **年齢系**: 幼女、少年少女、青年、中年、老人、etc.
- **種族系**: 人間、エルフ、獣人、ロボット、妖精、etc.

**実装内容**:
- テンプレート一覧UI（カテゴリ別に整理）
- テンプレートプレビュー機能
- 選択後に名前や詳細をカスタマイズ
- 複数テンプレートの組み合わせも可能（例: ツンデレ + 魔法使い）

**テンプレート例**:
```json
{
  "name": "ツンデレテンプレート",
  "category": "性格系",
  "definition": {
    "personality": "普段はそっけないが、本当は優しい。素直になれない性格",
    "speakingStyle": "「べ、別に〜」「〜なんだからね！」といった照れ隠しの口調",
    "firstPerson": "私",
    "secondPerson": "あんた",
    "catchphrases": ["べ、別にあんたのためじゃないんだからね！", "勘違いしないでよね"],
    "background": ""
  }
}
```

**メリット**:
- よくあるキャラクタータイプを簡単に作成できる
- キャラクター作成の時間短縮
- 初心者でも魅力的なキャラが作れる

**想定ユーザー**: 定番のキャラクタータイプで遊びたいユーザー

**実装の選択肢**:
1. ハードコードでテンプレートを用意
2. JSONファイルで管理（追加・編集が容易）
3. ユーザーがカスタムテンプレートを作成・共有できる機能

1. **ユーザーニーズの明確さ** - 実際に要望があるか
2. **実装コスト** - 開発にかかる時間とリソース
3. **既存機能との親和性** - 既存コードとの統合しやすさ
4. **インパクト** - ユーザー体験への影響度
5. **保守性** - 維持管理のしやすさ
